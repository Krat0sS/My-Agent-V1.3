## 🔧 修复

### 致命修复：技能匹配系统完全失效
- **根因**：`skills/loader.py` 的 `_extract_keywords()` 用正则 `[\u4e00-\u9fff]+` 做分词，把整个中文短语当成一个"词"，导致所有技能匹配分数为 0.0
- **修复**：改用 jieba 中文分词，正确拆分中文短语
- **修复**：`core/intent_router.py` 的 `match_skill()` 用户输入也改用 jieba 分词
- **新增同义词扩展**：自动为关键词添加同义词（如"整理"↔"清理"↔"收拾"↔"乱"），提高匹配覆盖率
- **调整评分权重**：命中率 70% + Jaccard 30%（原来 60/40），短输入不再被稀释
- **降低匹配阈值**：从 0.5 降到 0.4，让更多边界情况命中

### app.py 重写：零配置启动
- **问题**：启动后需要手动点"保存"和"应用配置"两个按钮才能用，体验差
- **修复**：
  - 使用 `@st.cache_resource` 自动初始化 ToolRegistry + 技能，启动即加载
  - API Key 填一次自动保存到 `.env`，下次启动自动读取
  - 工具和技能列表不依赖 API Key，启动即可查看
  - 发送消息时才检查 Key，没填则友好提示
  - `run_agent()` 每次调用前动态 reload config + 重置 LLM 客户端，确保用最新配置

### 工程修复
- **`.gitignore` 补全**：加入 `launch_log.txt`、`.DS_Store`、`Thumbs.db`、`*.swp`
